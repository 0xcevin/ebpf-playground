# Release v0.3.3 — 修复 CI arm64 编译与兼容模式 execve fallback

**发布日期**：2026-04-24  
**标签**：[v0.3.3](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.3.3)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 的 release 与 debug 版本。

---

## 一句话总结

修复 v0.3.2 的 CI 编译失败：兼容模式的 kprobe fallback 代码在 arm64 上因缺少 `struct user_pt_regs` 而无法编译；现改用架构内联寄存器访问，同时支持 amd64 与 arm64。

---

## 修复

### 1. 兼容模式 execve 追踪失败（CentOS 7 等老内核）

在 CentOS 7 (3.10.0-957.el7.x86_64) 等内核上，`syscalls:sys_enter_execve` tracepoint 不存在，导致 `-execve` 无输出。

- `bpf/trace_legacy.bpf.c` 新增 `kprobe/sys_execve` 与 `kretprobe/sys_execve`。
- `legacy.go` 在 tracepoint attach 失败后自动 fallback 到 `link.Kprobe` / `link.Kretprobe`。

### 2. CI arm64 编译失败

v0.3.2 将 legacy 的 `go:generate` target 改为 `amd64`，导致 `GOARCH=arm64` 构建时缺少 `loadTrace_legacy` 符号。

- `main.go`：将 legacy 的 `go:generate` target 恢复为 `amd64,arm64`，使 bpf2go 同时生成双架构对象。
- `bpf/trace_legacy.bpf.c`：不再使用 `bpf_tracing.h` 的 `PT_REGS_PARM1` / `PT_REGS_RC`（arm64 版本依赖 `struct user_pt_regs`，而 `vmlinux.h` 为 x86_64 导出）。改为根据 `__TARGET_ARCH_x86` / `__TARGET_ARCH_arm64` 直接内联寄存器访问。

## 完整变更列表

- `fix(bpf/trace_legacy.bpf.c)`：新增 kprobe fallback；使用架构内联宏替代 `PT_REGS_*`，支持 x86_64 与 arm64
- `fix(legacy.go)`：execve tracepoint attach 失败时自动 fallback 到 kprobe/kretprobe
- `fix(main.go)`：legacy `go:generate` target 改为 `amd64,arm64`，修复 arm64 缺失符号
- `build`：重新生成 `trace_legacy_x86_bpfel.go` / `.o` 与 `trace_legacy_arm64_bpfel.go` / `.o`
