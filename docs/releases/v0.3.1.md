## 一句话总结

将现代模式的连接级流量累加与预警功能完整移植到兼容模式（内核 < 5.8），实现双模式统一支持。

---

## 新增功能

### 1. 兼容模式流量统计

`bpf/trace_legacy.bpf.c` 完整移植现代模式的流量累加与预警逻辑：

- 新增 `conn_stats_map`（HASH）：以 `pid+fd` 为 key，累加 `rx_bytes` / `tx_bytes`。
- 新增 `io_enter_args`（HASH）：用于 write/read/sendto/recvfrom 的 enter/exit 配对。
- 新增 `config_map`（ARRAY）：存储 `-flow-threshold-mb` 设定的阈值。
- 覆盖 `write` / `read` / `sendto` / `recvfrom` / `close` 五组 tracepoint。
- 使用 `bpf_perf_event_output` 替代 ringbuf 发送 `FLOW` 预警事件。

### 2. 预警行为与现代模式一致

```bash
# 在兼容模式内核上同样生效
sudo ./ebpf-tracepoint -net -flow-threshold-mb 10
```

输出格式与现代模式完全相同：

```
FLOW     845804 curl             ALERT=RX RX=1.00MB TX=0.09KB 90.130.70.73:80
FLOW     1135   AliYunDunMonito  ALERT=TX RX=3.46KB TX=1.00MB 100.100.188.188:443
```

### 3. 实现细节差异

| 特性 | 现代模式 | 兼容模式 |
|------|---------|---------|
| 事件通道 | `BPF_MAP_TYPE_RINGBUF` | `BPF_MAP_TYPE_PERF_EVENT_ARRAY` |
| 事件发送 | `bpf_ringbuf_reserve` / `bpf_ringbuf_submit` | `bpf_perf_event_output(ctx, &pb, ...)` |
| 内存读取 Helper | `bpf_probe_read_user` | `bpf_probe_read` |
| 单条事件大小 | 312 字节 | 312 字节 |
