# Release v0.2.3 — 修复 CentOS 7 上兼容模式 Map BTF 加载失败

**发布日期**：2026-04-23  
**标签**：[v0.2.3](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.2.3)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 静态编译版本。

---

## 一句话总结

修复 CentOS 7 (3.10) 上兼容模式因内核不支持 Map BTF 而导致 map 创建失败的问题。

---

## 修复详情

### Bug 现象

在 CentOS 7.5 (内核 3.10.0-862.3.2.el7.x86_64) 上运行 v0.2.2：

```
[MODE] 内核 < 5.8，启用兼容模式 (perf buffer)
[FAIL] 加载 eBPF 兼容对象失败: field TracepointSysEnterAccept4: 
  program tracepoint__sys_enter_accept4: map accept4_sockaddr: 
  load BTF: detect support for Map BTF (Var/Datasec): 
  detect support for BTF: function not implemented
```

### 根因分析

`cilium/ebpf` 在创建 map 时，如果 `MapSpec.Key` 或 `MapSpec.Value` 不为 nil，会尝试调用 `btf.MarshalMapKV` 来序列化 BTF 类型信息。该函数内部检测内核是否支持 `BPF_MAP_TYPE_*` 的 BTF k/v（通过 `bpf(BTF_LOAD)`）。CentOS 7 的 3.10 内核没有 BTF 支持，系统调用返回 `ENOSYS`（function not implemented），导致加载失败。

### 修复方案

在 `legacy.go` 加载兼容对象前，清空所有 map 的 BTF 类型字段：

```go
for _, m := range spec.Maps {
    m.Key = nil
    m.Value = nil
}
```

这样 `cilium/ebpf` 在 `createMap` 时跳过 BTF 分支，以传统方式（仅使用 `key_size` / `value_size`）创建 map，与 3.10 内核完全兼容。
