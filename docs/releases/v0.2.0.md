# Release v0.2.0 — 双模式内核自适应

**发布日期**：2026-04-23  
**标签**：[v0.2.0](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.2.0)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 静态编译版本。

---

## 一句话总结

一套二进制，同时覆盖 **CentOS 7 (3.10)** 到 **最新 6.x 内核**。运行时自动检测内核版本，≥ 5.8 走高性能 ringbuf 现代模式，< 5.8 自动降级到 perf buffer 兼容模式。

---

## 主要变更

### 1. 双模式内核自适应（核心特性）

| 维度 | 现代模式 (≥ 5.8) | 兼容模式 (< 5.8) |
|------|-----------------|-----------------|
| **事件传输** | `BPF_MAP_TYPE_RINGBUF` | `BPF_MAP_TYPE_PERF_EVENT_ARRAY` |
| **可移植性** | BTF / CO-RE | 传统结构体定义（`vmlinux.h`） |
| **内存读取** | `bpf_probe_read_user` / `_user_str` | `bpf_probe_read` / `_str` |
| **典型系统** | Ubuntu 22.04+、RHEL 9+、云服务器 | CentOS 7/8、Ubuntu 20.04 |

用户无需手动选择，程序启动时自动根据 `uname -r` 分发到对应实现。

### 2. 新增文件

- `bpf/trace_legacy.bpf.c` — 面向 4.x / CentOS 7 / CentOS 8 的 eBPF 兼容程序。
- `modern.go` — 抽取并隔离原 `main.go` 中 5.8+ 的 ringbuf 加载与消费逻辑。
- `legacy.go` — 4.x 兼容模式的 perf buffer 加载与消费逻辑。
- `trace_legacy_bpfel.go` / `trace_legacy_bpfeb.go` — 兼容版 eBPF 对象的 Go 绑定（自动生成）。

### 3. 架构重构

```
main.go          → 入口：环境检测 + 模式分发 + 公共解析逻辑
modern.go        → 现代模式：加载 trace.o，ringbuf 消费
legacy.go        → 兼容模式：加载 trace_legacy.o，perf 消费
```

`Event` 结构体、`printEvent`、`parseSockaddr` 两边完全一致，避免维护两份用户态解析代码。

### 4. 兼容性扩展

| 目标系统 | 典型内核 | v0.1.0 | **v0.2.0** |
|---------|---------|--------|-----------|
| CentOS 7 / RHEL 7 | 3.10 | ❌ 不支持 | ✅ **兼容模式** |
| CentOS 8 / RHEL 8 | 4.18 | ❌ 不支持 | ✅ **兼容模式** |
| Ubuntu 20.04 | 5.4 | ❌ 不支持 | ✅ **兼容模式** |
| Ubuntu 22.04+ / RHEL 9+ | 5.15+ / 5.14+ | ✅ 现代模式 | ✅ 现代模式 |
| 云厂商定制内核 | 5.8 ~ 6.x | ✅ 现代模式 | ✅ 现代模式 |

## 使用示例

### 编译

```bash
make build
```

### 运行（自动选择模式）

```bash
# 5.8+ 内核示例输出
sudo ./ebpf-tracepoint
# [MODE] 内核 >= 5.8，启用现代模式 (ringbuf + BTF)

# 4.x 内核示例输出
sudo ./ebpf-tracepoint
# [MODE] 内核 < 5.8，启用兼容模式 (perf buffer)
```

### 选择性追踪

```bash
sudo ./ebpf-tracepoint -execve   # 仅追踪 execve
sudo ./ebpf-tracepoint -net      # 仅追踪网络 (connect, accept4)
```

## 升级指南（从 v0.1.0）

v0.2.0 **完全向后兼容**：
- 5.8+ 内核上的行为与 v0.1.0 完全一致（现代模式）。
- 新增的兼容模式不会影响到现有现代模式用户的任何功能或性能。

直接替换二进制即可，无需修改任何配置或命令行参数。

---

## 已知限制

1. **兼容模式无 BTF**：`trace_legacy.bpf.c` 依赖 `vmlinux.h` 中的结构体定义。若目标内核的 tracepoint 上下文结构体与编译机差异较大（极罕见），可能出现字段偏移偏差。tracepoint 的 `args[]` ABI 在 3.10 ~ 6.x 之间高度稳定，实际风险很低。
2. **CentOS 7 的 3.10 内核**：红帽对 eBPF 有大量 backport，tracepoint 和 perf buffer 通常可用，但某些极度精简的内核配置可能缺失 eBPF 基础支持。
3. **二进制体积**：由于打包了两套 eBPF 对象（现代 + 兼容，各大小端两份），v0.2.0 的二进制比 v0.1.0 略大（约 +1.5 MB）。
