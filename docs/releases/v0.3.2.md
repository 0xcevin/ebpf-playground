# Release v0.3.2 — 兼容模式 execve kprobe fallback

**发布日期**：2026-04-24  
**标签**：[v0.3.2](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.3.2)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 的 release 与 debug 版本。

---

## 一句话总结

修复 CentOS 7 等老内核（3.10）上兼容模式无法追踪 execve 的问题：当 syscall tracepoint 不可用时，自动回退到 kprobe/kretprobe 挂载。

---

## 修复

### 1. 兼容模式 execve 追踪失败

在 CentOS 7 (3.10.0-957.el7.x86_64) 等内核上，`/sys/kernel/debug/tracing/events/syscalls/sys_enter_execve` 不存在，导致 `link.Tracepoint` 挂载失败，`-execve` 模式无任何输出。

**修复**：在 `bpf/trace_legacy.bpf.c` 中新增 `kprobe/sys_execve` 与 `kretprobe/sys_execve`，通过 `pt_regs` 直接读取 `filename`（`di` 寄存器）和返回值（`ax` 寄存器）。`legacy.go` 在 tracepoint attach 失败后自动 fallback 到 kprobe/kretprobe，事件格式与 tracepoint 版完全一致。

## 完整变更列表

- `fix(bpf/trace_legacy.bpf.c)`：新增 `kprobe__sys_execve` / `kretprobe__sys_execve`，支持无 syscall tracepoint 的内核
- `fix(legacy.go)`：execve tracepoint attach 失败时自动 fallback 到 `link.Kprobe` / `link.Kretprobe`
- `chore(main.go)`：调整 legacy 的 `go:generate` target 为 `amd64`，使 bpf2go 正确生成 x86 架构宏
- `build`：重新生成 `trace_legacy_x86_bpfel.go` / `.o`
