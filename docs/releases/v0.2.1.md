# Release v0.2.1 — 修复兼容模式在无 BTF 内核上的加载失败

**发布日期**：2026-04-23  
**标签**：[v0.2.1](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.2.1)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 静态编译版本。

## 一句话总结

修复 v0.2.0 中兼容模式在 **Oracle Linux 8 (UEK 5.4.x)、CentOS 7/8** 等无 BTF 内核上加载失败的 bug。

---

## 修复详情

### Bug 现象

在 Oracle Linux 8 (内核 5.4.17-2102.201.3.el8uek.x86_64) 上运行 v0.2.0 兼容模式时：

```
[MODE] 内核 < 5.8，启用兼容模式 (perf buffer)
[FAIL] 加载 eBPF 兼容对象失败: field TracepointSysEnterAccept4: 
  program tracepoint__sys_enter_accept4: 
  apply CO-RE relocations: load kernel spec: 
  no BTF found for kernel version 5.4.17-2102.201.3.el8uek.x86_64: not supported
```

### 根因分析

`bpf2go` 编译 `trace_legacy.bpf.c` 时，`vmlinux.h` 中包含：

```c
#pragma clang attribute push (__attribute__((preserve_access_index)), apply_to = record)
```

该属性导致 clang 在 `.BTF.ext` 段中生成 **CO-RE 重定位记录**。`cilium/ebpf` 在加载程序时检测到这些重定位记录，会尝试从内核加载 BTF 规范来进行重定位。但目标内核没有 BTF，因此加载失败。

### 修复方案

为兼容版 `go:generate` 指令添加编译宏 `-DBPF_NO_PRESERVE_ACCESS_INDEX`：

```go
//go:generate go run github.com/cilium/ebpf/cmd/bpf2go -cc clang \
//  -cflags "-O2 -Wall -Werror -I/usr/include -I./bpf -DBPF_NO_PRESERVE_ACCESS_INDEX" \
//  -target bpfel,bpfeb trace_legacy bpf/trace_legacy.bpf.c
```

`vmlinux.h` 中的 `#pragma` 被 `#ifndef BPF_NO_PRESERVE_ACCESS_INDEX` 保护，因此该宏会禁用 `preserve_access_index` 属性。

### 验证结果

- `.BTF.ext` 中不再包含 `core_reloc` 段（仅保留 `func_info` 和 `line_info`）。
- `.BTF` 段完整保留，`cilium/ebpf` 仍可正常解析 BTF-style map 定义。
- 在当前 6.8 内核上测试：`loadTrace_legacy` + `LoadAndAssign` 成功。
