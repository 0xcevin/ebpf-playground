## 一句话总结

修复 CentOS 7 (3.10) 上 `rlimit.RemoveMemlock()` 因 cgroup 检测不支持的系统调用而失败的问题。

---

## 修复详情

### Bug 现象

在 CentOS 7.5 (内核 3.10.0-862.3.2.el7.x86_64) 上运行 v0.2.1：

```
[FAIL] 无法解除 memlock 限制: unexpected error detecting memory cgroup accounting: function not implemented
```

### 根因分析

`cilium/ebpf` 的 `rlimit.RemoveMemlock()` 在内部尝试检测 memory cgroup 会计功能时，调用了当前内核不支持的系统调用。CentOS 7 的 3.10 内核虽然支持基本的 rlimit 调整，但缺少 `RemoveMemlock` 依赖的某些较新接口。

### 修复方案

当 `rlimit.RemoveMemlock()` 失败时，降级到手动设置 `RLIMIT_MEMLOCK`：

```go
if err := rlimit.RemoveMemlock(); err != nil {
    // 降级：老内核手动设置 rlimit
    var rlim unix.Rlimit
    unix.Getrlimit(unix.RLIMIT_MEMLOCK, &rlim)
    rlim.Cur = rlim.Max
    unix.Setrlimit(unix.RLIMIT_MEMLOCK, &rlim)
}
```
