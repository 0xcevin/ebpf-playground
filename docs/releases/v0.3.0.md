## 新增功能

### 1. 动态缓冲区配置（`-buffer-level`）

新增命令行参数 `-buffer-level`，支持三级缓冲区大小，无需重新编译 eBPF 对象即可调节：

| level | 现代模式 (ringbuf) | 兼容模式 (perf buffer / CPU) | 适用场景 |
|-------|-------------------|---------------------------|---------|
| 1 | 256 KB | 16 KB | 开发测试、低负载 |
| 2 (默认) | 1 MB | 64 KB | 通用生产环境 |
| 3 | 4 MB | 256 KB | 高频网络追踪 |

现代模式的 ringbuf 是**全局共享**的（单实例），兼容模式的 perf buffer 是**每 CPU 独立**的（总量 = 单 CPU 大小 × 核数）。

### 2. 连接级流量统计与预警（现代模式）

新增 `-flow-threshold-mb` 参数，当单条 socket 连接的收发流量超过阈值时，实时输出 `FLOW` 预警事件。

**预警输出示例**：

```
FLOW     845804 curl             ALERT=RX RX=1.00MB TX=0.09KB 90.130.70.73:80
FLOW     1135   AliYunDunMonito  ALERT=TX RX=3.46KB TX=1.00MB 100.100.188.188:443
```

**实现原理**：
- `connect` / `accept4` 建立连接时，在 eBPF map（`conn_stats_map`）中创建统计条目（key = pid+fd）。
- 通过 `write` / `read` / `sendto` / `recvfrom` 的 enter/exit 配对，累加实际收发字节数。
- `close` 时自动清理 map 条目，防止内存泄漏。
- 超过阈值时通过 ringbuf 发送一次 `FLOW` 事件（带 `warned` 标志避免重复上报）。

**限制**：
- 仅现代模式（内核 ≥ 5.8）支持。
- `sendfile` / `splice` 等零拷贝路径不会被计入。
