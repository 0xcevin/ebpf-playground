## 一句话总结

在启动阶段自动检测 Kernel Lockdown、SELinux、perf_event_paranoid 等 RHEL 9 / AlmaLinux 9 常见的 eBPF 限制因素，并在 attach 失败时给出具体修复命令，显著降低权限报错的排查成本。

---

## 新增

### 1. 环境安全约束检测（`main.go`）

启动时自动扫描三类常见限制：

- **Kernel Lockdown**：读取 `/sys/kernel/security/lockdown`，若当前状态为 `integrity` 或 `confidentiality`（通常由 Secure Boot 触发），提示关闭 Secure Boot 或添加 `lockdown=0`。
- **SELinux**：调用 `getenforce`，若为 `Enforcing`，提示 `sudo setenforce 0` 临时测试。
- **perf_event_paranoid**：读取 `/proc/sys/kernel/perf_event_paranoid`，若 `>= 2`，提示设置为 `-1` 或 `0`。

检测到的每一项都会在启动日志中列出对应修复方案，方便用户直接复制执行。

### 2. attach 失败友好提示（`modern.go` / `legacy.go`）

当 `link.Tracepoint` 返回 `permission denied` 时，程序不再只输出裸错误，而是自动追加三条最可能的根因和命令：

```
Warning: failed to attach sys_exit_execve: cannot create bpf perf link: permission denied
  Hint: 在 RHEL 9 / AlmaLinux 9 / Rocky 9 等系统上，这通常由以下原因导致：
        1. Secure Boot 触发 Kernel Lockdown → 进 BIOS 关闭 Secure Boot
        2. SELinux Enforcing → sudo setenforce 0（临时测试）
        3. perf_event_paranoid 过高 → echo -1 | sudo tee /proc/sys/kernel/perf_event_paranoid
```

## 完整变更列表

- `feat(main.go)`：新增 `checkSecurityConstraints()`，启动时检测 lockdown / SELinux / perf_event_paranoid 并给出修复建议
- `feat(modern.go)`：attach 失败且错误含 `permission denied` 时，追加 RHEL 9 排查提示
- `feat(legacy.go)`：同上，兼容模式也享受同样的错误提示增强
- `build`：编译验证通过，无需重新生成 eBPF 对象（未修改 C 代码）

## 兼容性

| 模式 | 内核要求 | execve 追踪 |
|------|---------|------------|
| 现代模式 | ≥ 5.8 | syscall tracepoint ✅ |
| 兼容模式 | 3.10+ (RHEL 回移植) / 4.7+ | tracepoint + kprobe fallback ✅ |

> 本版本仅修改 Go 侧代码，未改动 `bpf/*.bpf.c`，两个模式的 `struct event` 大小保持一致，现代/兼容模式的对象文件无需重新生成。
