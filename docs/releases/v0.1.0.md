# Release v0.1.0 — 初始版本

**发布日期**：2026-04-23  
**标签**：[v0.1.0](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.1.0)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 静态编译版本。

---

## 一句话总结

基于 eBPF tracepoint 的轻量级系统调用追踪工具初始版本，支持监控 `execve`、`connect`、`accept4`。

---

## 核心功能

- **eBPF tracepoint 监控**：监控 `execve`（进入/退出）、`connect`、`accept4` 三类核心系统调用。
- **事件传输**：使用 `BPF_MAP_TYPE_RINGBUF` 进行内核-用户态事件传输。
- **CO-RE 方案**：采用 Compile Once – Run Everywhere，一个静态二进制可在同架构不同 5.8+ 内核间分发运行。
- **选择性追踪**：新增 `-execve` 和 `-net` 命令行参数，可单独追踪 execve 或网络相关系统调用。
- **环境自检**：启动时自动检测内核版本、BTF 可用性、root 权限、memlock 限制，并给出明确的中文提示。
- **优雅降级**：单个 tracepoint attach 失败不再导致程序退出，而是打印 Warning 并继续挂载其他可用事件。
- **GitHub Actions Release CI**：`.github/workflows/release.yml`，推送 `v*` tag 自动编译 `linux/amd64` 与 `linux/arm64` 静态二进制并发布 Release。
