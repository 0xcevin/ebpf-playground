# Release v0.2.6 — 构建元数据嵌入与 Debug 二进制

**发布日期**：2026-04-23  
**标签**：[v0.2.6](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.2.6)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 的 release 与 debug 两个版本。

---

## 一句话总结

在二进制中嵌入编译时元数据（版本、时间、内核、Go 版本等），并在 Release CI 中同时发布 release 与 debug 两种二进制。

---

## 新增功能

### 1. 构建元数据嵌入

`main.go` 新增以下编译期变量，通过 `-ldflags -X` 在构建时注入：

| 变量 | 说明 |
|------|------|
| `Version` | 程序版本（如 `v0.2.6`） |
| `BuildTime` | 编译时间戳 |
| `BuildOS` / `BuildArch` | 编译环境操作系统与架构 |
| `BuildKernel` | 编译机内核版本 |
| `GoVersion` | 使用的 Go 版本 |

启动时自动打印：

```
版本: v0.2.6
编译时间: 2026-04-23T18:40:00+08:00
编译环境: linux/amd64
编译内核: 6.8.0-63-generic
Go 版本: go1.24.0
```

### 2. Debug 二进制

Makefile 新增 `build-debug` 目标，编译时**不**使用 `-s -w` 剥离符号表，保留调试信息：

```bash
make build-debug
# 输出: ebpf-tracepoint-debug
```

Release CI 现在会同时上传两种二进制到 GitHub Release：
- `ebpf-tracepoint`：release 版（`-s -w` 剥离，体积小）
- `ebpf-tracepoint-debug`：debug 版（保留符号表，便于 `gdb` / `delve` 调试）
