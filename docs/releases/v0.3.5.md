# Release v0.3.5 — 修复 RHEL 9.4 / AlmaLinux 9.4 上 attach permission denied

**发布日期**：2026-04-24  
**标签**：[v0.3.5](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.3.5)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 的 release 与 debug 版本。

---

## 一句话总结

降级 `cilium/ebpf` 至 v0.8.1，强制使用旧式 `ioctl` attach 替代 `BPF_LINK_CREATE`，彻底解决 RHEL 9.4 / AlmaLinux 9.4 / Rocky 9 上 `cannot create bpf perf link: permission denied` 的问题。

---

## 修复

### RHEL 9.4 上 tracepoint attach 失败

在 AlmaLinux 9.4（`5.14.0-427.20.1.el9_4`）等系统上，`cilium/ebpf v0.21.0` 的 `link.Tracepoint` 优先使用内核 `BPF_LINK_CREATE`（5.15+ backport）来 attach tracepoint。RHEL 9.4 的 kernel lockdown / SELinux / perf_event 策略在该系统调用上返回 `EPERM`，且 `cilium/ebpf` 不会自动回退到旧式 attach。

- `go.mod`：`github.com/cilium/ebpf v0.21.0 → v0.8.1`
- `modern.go` / `legacy.go`：适配 v0.8.1 API，`link.Tracepoint` / `link.Kprobe` / `link.Kretprobe` 去掉 opts 参数
- `legacy.go`：v0.8.1 的 `ebpf.MapSpec` 不含 `Key`/`Value` 字段，改用反射兼容清空 BTF 的逻辑

v0.8.1 的 `link.Tracepoint` 使用 `perf_event_open` + `ioctl(PERF_EVENT_IOC_SET_BPF)` 旧方式 attach，不受 `BPF_LINK_CREATE` 权限限制影响。

## 完整变更列表

- `fix(go.mod)`：降级 `github.com/cilium/ebpf` 至 `v0.8.1`
- `fix(modern.go)`：适配 v0.8.1 `link.Tracepoint` API（3 参数，移除 opts）
- `fix(legacy.go)`：适配 v0.8.1 `link.Tracepoint` / `link.Kprobe` / `link.Kretprobe` API；使用反射兼容 `MapSpec.Key/Value` 字段缺失
- `build`：编译验证通过，无需重新生成 eBPF 对象（未修改 C 代码）
