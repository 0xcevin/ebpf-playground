# Release v0.3.6 — 修复 CI 编译失败（bpf2go 版本锁定）

**发布日期**：2026-04-24  
**标签**：[v0.3.6](https://github.com/0xcevin/ebpf-playground/releases/tag/v0.3.6)  
**二进制下载**：GitHub Release 页面提供 `linux/amd64` 与 `linux/arm64` 的 release 与 debug 版本。

---

## 一句话总结

在 v0.3.5 的基础上修复 CI 编译失败：`cilium/ebpf` 降级至 v0.8.1 后，其内置 `bpf2go` 不支持 `amd64,arm64` 多 target，现通过锁定 `bpf2go@v0.21.0` 保证代码生成与 CI 构建正常。

---

## 修复

### CI 编译失败（v0.3.5）

v0.3.5 降级 `cilium/ebpf` 至 v0.8.1 后，CI 中 `go generate ./...` 调用的是 v0.8.1 的 `bpf2go`，该版本：
- 不支持 `-target amd64,arm64`（仅支持 `bpfel,bpfeb`）
- 生成的文件名不含架构后缀，与 `//go:embed` 不匹配

修复方式：
- `main.go` 的 `go:generate` 注释固定使用 `github.com/cilium/ebpf/cmd/bpf2go@v0.21.0`
- 运行时库仍为 `v0.8.1`，代码生成工具与运行时库解耦

---

## 完整变更列表（相对于 v0.3.4）

- `fix(go.mod)`：降级 `github.com/cilium/ebpf` 至 `v0.8.1`
- `fix(modern.go)`：适配 v0.8.1 `link.Tracepoint` API（3 参数，移除 opts）
- `fix(legacy.go)`：适配 v0.8.1 `link.Tracepoint` / `link.Kprobe` / `link.Kretprobe` API；使用反射兼容 `MapSpec.Key` / `MapSpec.Value` 字段缺失
- `fix(main.go)`：`go:generate` 锁定 `bpf2go@v0.21.0`，修复 CI 中多架构代码生成失败
- `build`：编译验证通过，无需重新生成 eBPF 对象
